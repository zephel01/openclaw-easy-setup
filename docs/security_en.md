## Overview

openclaw-easy-setup is designed with a "security-by-default" philosophy. All security features are enabled out of the box and require explicit action to weaken.

## Security Layers

```

```

## 1. Network Security

### Port Binding

All ports bind to `127.0.0.1` (localhost). They cannot be accessed from LAN or external networks.

```yaml
# docker-compose.yml
ports:
  - "127.0.0.1:18789:18789"  # OpenClaw Gateway
  - "127.0.0.1:11434:11434"  # Ollama
```

### Firewall Configuration

As an additional defense layer:

**Linux (UFW):**
```bash
sudo ufw deny 18789/tcp
sudo ufw deny 11434/tcp
```

**macOS (pf):**
```bash
echo "block in on en0 proto tcp from any to any port 18789" | sudo pfctl -ef -
```

### Docker Network

Inter-container communication uses a dedicated bridge network (`openclaw-net`) with subnet `172.28.0.0/16`, isolated from the host network.

## 2. Container Security

### OpenClaw Container Hardening

| Setting | Value | Effect |
|---------|-------|--------|
| `security_opt: no-new-privileges` | `true` | Prevents `setuid`/`setgid` privilege escalation |
| `read_only` | `true` | Container filesystem is immutable |
| `cap_drop` | `ALL` | All Linux capabilities removed |
| `cap_add` | `NET_BIND_SERVICE` | Only low-port binding allowed |
| `tmpfs` | `/tmp:256M`, `/run:64M` | Temp files constrained to memory |
| `USER` | `openclaw` (non-root) | Runs as unprivileged user |

### Dockerfile Security

Multi-stage build minimizes image size and attack surface:

- **Build stage**: Node.js + npm install OpenClaw
- **Runtime stage**: Minimal deps only (ca-certificates, curl, tini)
- **Tool removal**: Package managers (apt, dpkg) removed from final image
- **tini**: PID 1 for proper signal handling and zombie reaping

### Resource Limits

```yaml
deploy:
  resources:
    limits:
      memory: 2G    # OpenClaw
      cpus: "2.0"
    # Ollama
    limits:
      memory: 8G
      cpus: "4.0"
```

## 3. OpenClaw Application Security

### Authentication

```yaml
gateway:
  auth:
    mode: "token"
    token: "<auto-generated-64-char-hex>"
```

- Token generated via `openssl rand -hex 32` (256-bit entropy)
- Stored in `.env` file (permissions: 600)
- Disabling requires explicit `mode: "none"` setting

### DM Policy

| Policy | Behavior |
|--------|----------|
| `pairing` (default) | First-time DMs require approval. Gates unknown senders |
| `open` | Accepts all DMs (not recommended) |

### Tool Restrictions

```yaml
tools:
  deny:
    - exec      # Shell command execution
    - browser   # Browser automation
    - cron      # Scheduled execution
```

These are denied by default. Modify `OPENCLAW_DENY_TOOLS` in `config.env` to change.

### Sandbox Isolation

```yaml
security:
  sandboxInheritEnv: false
```

Host environment variables are not passed to AI agent sandboxes, preventing API key or system info leakage through agents.

## 4. Credential Management

### Docker Environment (.env)

```
.env                  # Permissions: 600
├── ANTHROPIC_API_KEY
├── OPENAI_API_KEY
├── OPENCLAW_AUTH_TOKEN
└── ...
```

- Listed in `.gitignore` — never committed to Git
- `setup.sh` applies `chmod 600` automatically

### Native Environment (~/.openclaw/)

```
~/.openclaw/           # Permissions: 700
├── .env               # Permissions: 600 — Basic config (use secrets/ for API keys)
└── openclaw.json      # Permissions: 600 — Config with token
```

### ClawX Keychain Integration

ClawX stores API keys in the OS native keychain:

| OS | Storage |
|----|---------|
| macOS | macOS Keychain |
| Windows | Windows Credential Manager |
| Linux | libsecret |

Encryption: AES-256-GCM with separate table and per-row keys.

## 5. Security Checklist

### Required

- [ ] `.env` file permissions are `600`
- [ ] `docker compose ps` shows healthy containers
- [ ] Ports 18789/11434 are not externally accessible
- [ ] `./setup.sh --doctor` reports no errors

### Recommended

- [ ] OS firewall blocks the ports
- [ ] Dangerous tools (exec, browser, cron) in deny list
- [ ] `dmPolicy` set to `pairing`
- [ ] `requireMention` set to `true`
- [ ] Docker resource limits appropriate for your environment

### Periodic

- [ ] Update Docker images regularly (security patches)
- [ ] Rotate API keys
- [ ] Review logs: `docker compose logs openclaw`

## 6. Weakening Security (Use with Caution)

### Disable Auth (Dev only)

```env
OPENCLAW_AUTH_MODE=none
```

> **Never disable in production.**

### Relax Tool Restrictions

```env
OPENCLAW_DENY_TOOLS=browser
# Allows exec and cron, still denies browser
```

### Allow External Access (Not recommended)

Change port binding in `docker-compose.yml`:

```yaml
ports:
  - "0.0.0.0:18789:18789"
```

> Only use behind a reverse proxy (nginx/Caddy) with TLS.

## 7. Incident Response

### If API Keys Are Compromised

1. Immediately revoke the key in the provider's dashboard
2. Set the new key: `echo "new-key" > secrets/PROVIDER_API_KEY`
3. `docker compose restart openclaw`
4. Check Git history for accidental commits

### If Unauthorized Access Is Suspected

1. `docker compose down` to stop services
2. Capture logs: `docker compose logs openclaw > incident.log`
3. Regenerate auth token in `.env`
4. Restart services
